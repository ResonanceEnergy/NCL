Fix 1 - runtime/calendar/watchlist.py - broken scanner endpoint paths
  /scanner/goat -> /stocks/scanner/goat

  Calendar watchlist scanner_todos was silently 404ing both. Now flows.

Fix 2 - runtime/portfolio/moomoo_adapter.py - options multiplier P0
  Moomoo SDK reports cost_price as per-share premium. Options multiplier
  100 was missing -> cost_basis underrepresented 100x -> total_pl math
  was poisoned. Now: cost_basis = cost * qty * (100 if option else 1).
  Recompute unrealized_pl from MV - cost_basis (don't trust raw pl_val
  which Moomoo bug returns == market_value on options). Also fixes
  today_pl scaling and clamps absurd daily_pl_pct.
  New fields on position dict: cost_basis, multiplier.

Fix 3 - runtime/portfolio/portfolio_manager.py - _total_pl cost_basis fallback
  Was divide-by-zero when adapter didn't set cost_basis (every Moomoo
  position pre-fix) -> total_pl_pct always 0.0 despite total_pl having
  real value. Now derives cost_basis from avg_cost * qty * multiplier
  with sensible defaults (100 for options, 1 otherwise).

Fix 4 - runtime/portfolio/portfolio_manager.py - per-account double-count
  Moomoo's net_liquidation includes total assets (positions already in).
  SnapTrade/IBKR's net_liquidation is cash-only and needs positions
  added back. Previous code added positions to BOTH unconditionally ->
  Moomoo accounts showed 2.7x over-counted balances in iOS account
  switcher. Now broker-aware aggregation. Also added positions_count
  per account.

Fix 5 - runtime/stocks/scanner.py - GOAT R:R math
  Fixed +10/+15/+25% targets produced R:R < 1.0 on every high-vol name
  (where 2xATR stop > 5% drop). Stated strategy is 1:2 minimum.
  Now ATR-based: T1 = +3xATR, T2 = +5xATR, T3 = +7xATR.
  R:R now lands at 1.5R / 2.5R / 3.5R consistently.
