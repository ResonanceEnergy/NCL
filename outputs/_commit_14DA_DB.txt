# Wave 14DB — Watchlist always in Intel context

## Backend

NEW runtime/api/routers/watchlist.py (~110 LOC)
  GET  /watchlist/tickers — returns {tickers, tradingview_url, updated_at}
  POST /watchlist/tickers — replaces the list (normalizes uppercase
                            + dedups + strips whitespace)
  Persists to data/watchlist/tickers.json.

  Module-level helper get_watchlist_tickers() consumed by:
    runtime/awarebot/agent.py situational scorer.

runtime/api/routes.py
  Registers watchlist_router after the existing portfolio/paper/
  polymarket/calendar/feedback set.

runtime/awarebot/agent.py compute_situational_relevance
  NEW kwarg: tickers_in_watchlist: Optional[set[str]] = None
  When set and any ticker is mentioned in the signal text,
  situational score += 0.45. Slightly less than journal (0.5)
  since the watchlist is persistent intent vs journal's daily
  intent, but still strong enough to bubble watchlist hits
  consistently into NOW/HIGH route.

runtime/awarebot/agent.py _get_situational_ctx
  Loads watchlist tickers from disk on the same 30-min TTL as
  the rest of situational context. Populates ctx[
  "tickers_in_watchlist"].

runtime/awarebot/agent.py score loop (line ~2185)
  Threads tickers_in_watchlist through into the call AND, when a
  ticker matches, stamps signal.metadata["watchlist_hit"] =
  "<ticker>" so downstream consumers can highlight without
  re-scanning.

Live verified: POSTed 9 tickers (XRP/SLV/GLD/NVDA/GME/PLTR/MSFT/
AAPL/GRRR) → tickers.json written with updated_at stamp.

## iOS

WatchlistView settings sheet gains a TextEditor for the ticker
list (comma-separated). On Save:
  - Normalized client-side (uppercase, dedup, strip).
  - POSTed to /watchlist/tickers along with the URL.
  - Mirrored into @AppStorage so it survives offline restarts.
  - "Saved N ticker(s)" success message; "Save failed — …" on
    error.

WatchlistView.onAppear pulls the latest from the backend so
the iPhone and iPad converge if NATRIX edited from the other.

IntelNowView.IntelNowRow + IntelStreamView.IntelStreamRow
  NEW yellow WATCH badge ("WATCH · TICKER") when
  metadata.watchlist_hit is set. Placed first in the badge row
  so it visually dominates regardless of route_level color.
