============================================================
NOT BUILT: live auto-executor / order routing
============================================================
Hard line. System prompt: "never execute a trade, place an order, send
money, or initiate a transfer on the user's behalf - always ask the user
to perform those actions themselves." The audit-doc literature (Trading-
R1, Frontiers AI survey, QuantInsti) is unanimous: LLM-augmented systems
should keep order placement on the human side. The order_preview module
below is the safe substitute.

============================================================
order_preview.py (~210 LOC) — DRY-RUN ONLY
============================================================
preview_order() takes a proposed trade and returns a complete dry-run
report:
  - is_preview_only: True (always)
  - submission_blocked: True (always)
  - governor_decision: composed risk_governor check
  - wash_sale_flags: cross-account loss check (J4a)
  - lot_impact_preview: HIFO recommendation for sells (J4b)
  - estimated_commission_usd: from trade_cost_ledger median (J0a)
  - submission_payloads: per-broker IBKR/Moomoo/SnapTrade strings
    formatted for the OPERATOR to COPY-PASTE into the broker UI
  - Every payload starts with `# IS_PREVIEW_ONLY = True` sentinel

NCL has no credentials to submit and will never submit. Submission is
operator-only via the broker's own ticket. This makes the human-in-the-
loop pattern a single copy-paste instead of authoring orders from scratch.

============================================================
backtest_harness.py (~150 LOC)
============================================================
NOT a strategy R&D backtester. ONE thing: counter-factual replay of
trade_idea_tracker history through the CURRENT risk_governor config.
"What would my P&L look like if today's heat caps + drawdown bands had
been in place over the past N days?"

replay_window(lookback_days=90) reads trade_ideas.jsonl + snapshots.jsonl;
re-runs each idea through check_proposed_trade() with the NAV at issue
date. Outputs original_R vs replayed_R + per-strategy delta + blocked +
throttled lists. Tunes heat caps without flying blind.

============================================================
manual_adapter.py (~170 LOC)
============================================================
The audit said 6 brokers is saturated. True for live brokers — but cold
storage (hardware wallet Bitcoin), paper certificates, employer pre-IPO
stock, private placements all exist in real portfolios. Manual adapter
fits the same 7-method interface as the 6 broker adapters so it slots
in alongside them.

Storage: data/portfolio/manual_holdings.json (operator-maintained, file
edits or REST CRUD via /portfolio/manual/{positions,position,account}).
fetch_quotes() returns empty so the quote fallback chain takes over.

============================================================
quote_source.py (~160 LOC)
============================================================
Abstraction layer for the quote-fill chain (broker fast_info ->
yfinance -> cached -> None). NOT a new paid feed (audit said premature).
What this gives us: when the day comes that we wire Polygon/IEX, it's a
one-file change instead of touching every adapter.

QuoteSource protocol + StaticOverrideSource + YFinanceSource + CachedSource
+ QuoteChain composite with per-source stats (last_source, miss_count).
default_quote_chain() returns a process-wide singleton chain.

============================================================
streaming_scaffold.py (~150 LOC)
============================================================
NOT a live WebSocket subscriber. The BASE CLASS that an opt-in adapter
(IBKR updateEvent, Moomoo push) can subclass when J7c slippage data
shows the 60s polling lag matters.

PositionDeltaPublisher (abstract) + PositionDeltaConsumer (applies
deltas to cache + optional re-check callback) + MockDeltaPublisher for
tests. Zero live streams open today; opt-in only.
