Wave 14AW: Calgary events live via Eventbrite Canada JSON-LD

Per NATRIX directive 2026-05-30 — Edmonton + Calgary open-data
portals should give NATRIXs home cities native event feeds. Wave
14AH shipped Edmonton but Calgary was stubbed because data.calgary.ca
has no events dataset (verified again 2026-05-30 — open data portal
only has building permits, community development, environmental data).

WORKING FALLBACK: Eventbrite Canada all-events page for Calgary
at https://www.eventbrite.ca/d/canada--calgary/all-events/ embeds
an application/ld+json ItemList of ~20 upcoming Calgary events
directly in the HTML. No auth, no rate limit beyond polite use.

CHANGES:

runtime/intelligence/free_sources.py
  - Top of file: added missing `import json`. Was causing the prior
    Wave 14AH stub fetch_calgary_events to silently fail with
    NameError swallowed by the outer try/except. Other JSON-LD-using
    callers were unaffected because they used json indirectly via
    Socrata helpers (which use the existing httpx response.json()).
  - fetch_calgary_events rewritten as a real JSON-LD scrape. Uses
    httpx with a browser UA, regex-extracts script type
    application/ld+json blocks, finds ItemList, projects each Event
    to NCL row shape (title/date/end_date/venue/url/source/desc).
    Cached 2h. Window-filtered by today + days_ahead.

runtime/calendar/local_events.py
  - get_local_events: added a city_id == "calgary" branch that calls
    fetch_calgary_events for the same days-ahead window the Edmonton
    branch uses. Events tagged source=eventbrite.ca.
