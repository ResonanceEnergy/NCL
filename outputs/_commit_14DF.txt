# composeNightWatchMarkdown

Builds .md from the parsed brief in the same shape AM/PM/Executive
use:

  # Night Watch — YYYY-MM-DD

  **Status:** GREEN | YELLOW | RED | UNKNOWN

  ## KEY FINDINGS
  - finding 1
  - finding 2

  ## RECOMMENDATIONS
  - step=1 · action=...
  - step=2 · action=...

  ## SYSTEM HEALTH
  - health line 1
  - health line 2

  ## COST REPORT
  - cost line 1

  ## DIAGNOSTICS
  ### COST ANALYSIS
  <text content of rawAppendix["cost_analysis"]>
  ### LOG ANALYSIS
  <text content of rawAppendix["log_analysis"]>
  ...

  ## NO DATA  (only when ALL arrays are empty)
  Night Watch hasn't produced structured output for this brief yet.
  The 2am ET cycle may still be in progress, OR all Sonnet triage
  calls returned an error — check `/system/silent-failures` for
  `night_watch_triage_failed`.

The empty-state body specifically pinpoints the Wave 14CX
diagnostic path so NATRIX (or future Claude) can verify whether
the silent-failure counters caught the issue.

Recommendations get the `step=N · action=...` key=value treatment
so the iOS renderer stacks them in cyan monospace rows.

Rawappendix sections become H3 sub-sections under "DIAGNOSTICS"
H2 — keys upper-cased with underscores → spaces. Content preserves
its line structure since the analyst output is already loosely
formatted prose.

# Accent color = status color

  GREEN  → bright green H2 rule + cyan dots
  YELLOW → yellow rule
  RED    → red rule
  UNKNOWN → gray rule (status itself is the cue)

Matches the existing status pill so the page reads as one
coherent palette.
