# 4. Proper options schema (brief_council.py + brief_archiver.py)
   Chair prompt Rule 9 (NEW) mandates for type="options":
     option_strike (float)        — strike price
     option_dte (int)             — days to expiry
     option_right ("call"|"put")
     premium (float)              — entry premium per share ($2.50)
     underlying_entry (float)     — current underlying STOCK price
     underlying_target (float)    — target underlying price
     underlying_stop (float)      — invalidation on underlying
     max_risk = premium × 100 × contracts
     Legacy entry/stop/target MIRROR the underlying triple.

   Plus an escape hatch: "if you can't supply ALL options
   fields with real numbers, emit it as type='stock' using
   the underlying's price triple instead." → defeats the
   "fake options idea" failure mode.

   runtime/intelligence/brief_archiver.py re-enables options
   registration (Wave 14CT-v2 was a hard skip). New gate:
   only registers when ALL of underlying_entry / _stop /
   _target / option_strike / option_dte / option_right /
   premium are present. Otherwise skips with a precise log
   line that lets future debug see what was missing.
   The paper trade uses the UNDERLYING triple as
   entry/stop/target (paper engine speaks equity-style);
   option specifics are stashed in metadata for a future
   broker-paper-options layer.
