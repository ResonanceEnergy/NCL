Wave 14AJ: Local sentiment classification (FinBERT + twitter-roberta)

Per COST_AUDIT_OSS_ADDENDUM_2026-05-30 + FREE_RESOURCES Tier D
recommendation: replace ad-hoc Sonnet calls in AWAREBOT sentiment
scoring with two open-weight transformer models running on M1 Ultra MPS.

NEW MODULE: runtime/intelligence/local_sentiment.py
  - score_financial(text) -> ProsusAI/finbert
    Financial-news sentiment (Positive/Neutral/Negative). Use for
    headlines + SEC filings + earnings transcripts + Fed press releases.
  - score_social(text) -> cardiffnlp/twitter-roberta-base-sentiment-latest
    Social-media sentiment trained on 124M tweets. Use for Bluesky / X /
    Mastodon / Reddit / Telegram posts.
  - score_batch(texts, domain) -> efficient batched inference.

Lazy-loaded with thread-safe singleton pattern; first call downloads
weights to ~/.cache/huggingface/hub (~ 440 MB FinBERT + ~500 MB
Twitter-RoBERTa = 940 MB one-time cost). MPS-accelerated on M1 Ultra
when torch.backends.mps is available. ~5-20ms per inference vs
500-2000ms for an Anthropic API round-trip — and zero dollars per call.

Output shape (canonical NCL format):
  label: positive|neutral|negative
  score: 0.0-1.0 (confidence on top class)
  polarity: -1.0 to +1.0 (positive minus negative)
  raw: positive, neutral, negative probabilities
  model: finbert or twitter-roberta

Verified live (M1 Ultra, MPS device, first run with cold model
download then cached):
  FinBERT:
    "TSLA crushes earnings ... raises FY guide"  -> positive +0.93
    "NVDA misses on revenue, margins decline"    -> negative -0.96
    "Fed signals June hold; markets shrug"       -> negative -0.81
  Twitter-RoBERTa:
    "bitcoin breaking out hard"                  -> positive +0.69
    "this market makes no sense, im done lol"    -> negative -0.92
    "just hodling and ignoring the noise"        -> neutral

lets us swap to FinBERT2 the day it lands without code change.
