## 3. The Cross-Reference Engine (new module — the breakthrough mechanism)

**Module**: `runtime/cross_reference/` (~250 LOC, new)

### 3.1 Promotion rule

An AWAREBOT signal becomes a `PROMOTED_CANDIDATE` when ONE OF:

1. **Ticker convergence**: same ticker mentioned in ≥2 distinct AWAREBOT sources within last 4h
2. **Theme convergence**: same keyword cluster (LLM-extracted) in ≥3 distinct sources within last 24h
3. **News+Trends verifier hit**: signal-ticker shows up in News (RSS) AND Google Trends spike on same day (the "your intel is also in the press AND search" double-confirmation NATRIX described)

### 3.2 Pipeline

```
AWAREBOT writes signal to data/intelligence/agent_signals.jsonl
  ↓
cross_reference.scan() runs every 5min over last-4h window
  ↓
For each new signal, check rules 1-3 above
  ↓
If promoted:
  - Tag PROMOTED_CANDIDATE with sources list + convergence-strength
  - Append to data/cross_reference/promotions.jsonl
  - Notify TRADERAGENT via intel_request("awarebot.promoted_candidate", ...)
  - Surface in INTEL→NOW with "HOT 3×" badge
  ↓
TRADERAGENT receives → runs sanity gate (price ≥ $5, ADV ≥ $10M, 52w-range OK)
  ↓
If passes: writes to data/portfolio/auto_trader/organic_seeds.jsonl
            → next brief's executor sees this as ORGANIC IDEA SEED
  ↓
If fails: stays in INTEL tab tagged "hot but not actionable"
            → still visible to operator, just doesn't reach the trade chain
```

### 3.3 Why this fixes the noise problem

- Today: every HIGH-band signal pretends to be decision-grade
- New: AWAREBOT sources stay loose (they're entertainment), but **only the converged ones cross the lane border**. Most noise dies in INTEL, never reaches TRADERAGENT or the Brief
- The 60% HIGH-band saturation drops because the discrimination moved from "scored ≥ 0.55" to "confirmed by ≥ 2 independent sources"
