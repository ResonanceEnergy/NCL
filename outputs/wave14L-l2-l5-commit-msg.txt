L2 — QUANT SCANNER SUITE (runtime/portfolio/auto_trader/quant_scanners.py)

5 scanners running as ncl-auto-trader-quant-scan scheduler task
(30min market / 2hr off-hours). Each scanner:
  1. Calls capability_registry.check_and_request → if dep missing,
     auto-emits importance-95 tool:capability_request MemUnit + returns []
  2. Runs its quant logic against env-configurable WATCHLIST
     (default: SPY QQQ IWM NVDA TSLA AAPL MSFT AMD GOOG META AMZN
      SLV GLD XLE XLF XLK)
  3. Emits trade_ideas via trade_idea_tracker.record_emission with
     source="quant:<scanner_name>"

Scanners shipped:

  mean_reversion_scanner — RSI<14 + Bollinger lower-touch on uptrending
                            (price > 50d SMA). Long with stop 3% below
                            lower BB, target at mid BB.

  pead_scanner            — Post-earnings drift: ticker reported ≤5d ago
                            with +3%+ 1d ROC → long 20d drift, stop 7%,
                            target +20%.

  factor_scanner          — Long Leading-quadrant sector ETFs from
                            rotation_tracker with positive 5d rs_momentum.

  pairs_scanner           — 5 default cointegrated pairs (XLE/XOM, XLK/MSFT,
                            XLF/JPM, GLD/SLV, SPY/QQQ). 60-day price-ratio
                            z-score > 2 → long the cheap leg.

  whale_flow_scanner      — Stub awaiting UNUSUAL_WHALES_API_KEY. Calls
                            check_and_request → emits capability_request
                            on every tick until key is configured.

Each scanner has its own embedded RSI/Bollinger/z-score math
(no scipy dep). All yfinance fetches are asyncio.to_thread-wrapped to
keep the event loop alive.

L5 — CAPABILITY REGISTRY (runtime/portfolio/auto_trader/capability_registry.py)

The agent's self-awareness layer. 11 capabilities seeded:

  yfinance              free, import probe
  earnings_calendar     module probe → runtime.stocks.enrichments
  rotation_snapshot     file marker → data/rotation/{today}.json (2d max)
  cycle_phase           file marker → data/rotation/cycle-{today}.json
  ivr_data              soft signal, often unavailable
  unusual_whales        env required → UNUSUAL_WHALES_API_KEY
  finnhub               env required → FINNHUB_API_KEY
  ibkr_market_data      env + module → IBKR_HOST/PORT/CLIENT_ID + ib_insync
  polymarket            module probe → httpx
  coingecko             module probe → httpx
  fred                  optional FRED_API_KEY for some series

check_capability(name) is the hot-path API. Returns
{available, status, last_check_iso, gap_reason, missing_env, marker_age_days}
with status one of: ok / missing_env / missing_module / stale_marker /
missing_marker / unknown / disabled_check.

check_and_request(name, brain=..., requesting_module=...) is the
combined check + auto-MemUnit on gap. Dedupes per (capability, module,
ET date) so a stuck gap doesn't spam the memory store. Emitted MemUnit
has source="tool:capability_request" + tags including
capability:<name> and requested_by:<module>.

list_gaps() returns currently-unavailable capabilities for the iOS
"Capability Gaps" card.
