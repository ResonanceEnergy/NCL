============================================================
K0b — Entry-criteria policy (operator-tunable)
============================================================
runtime/portfolio/auto_trader/policy.py (~270 LOC)

AutoTraderPolicy dataclass — every field has a safe default + env-
overridable. Operator can PATCH any field via REST without bouncing
the brain (policy is loaded fresh each check).

Gates (defaults):
  require_governor_approved: True
  block_on_drawdown_halt: True       (K0c integration)
  block_on_breadth_veto: True
  require_stop_price/target/R_per_share: True
  require_source_citations: True
  min_R_R_ratio: 1.5                 (above PaperTradingEngine 1.0 floor)
  min_stop_distance_pct: 0.5         (no microscopic stops)
  max_stop_distance_pct: 15.0        (no irrational risk per share)
  require_thesis_min_chars: 20
  valid_stop_types: price|atr|volatility|time|thesis_break
  goat_require_with_trend: True      (counter-trend GOAT = operator-only)
  allow_counter_trend: False         (operator must opt in via PATCH)
  use_confidence_as_size_multiplier: True
                                      (research-supported: size by conf,
                                       don't gate on it)
  min_confidence_pct: 0
  max_opens_per_tick: 3
  max_opens_per_day: 12
  cooldown_seconds_after_open: 30

auto_open_eligible(idea, governor_decision) -> (bool, reason) is the
single entry point. Phase 2's loop will call this for every emitted
trade idea before opening a paper trade.

============================================================
K0c — Drawdown auto-pause wiring
============================================================
runtime/portfolio/auto_trader/state.py (~180 LOC)

AutoTraderState dataclass with explicit drawdown_halt_pause + band
fields. set_drawdown_halt(halted, band) is what the Phase 2 loop calls
each tick after reading drawdown_bucket.get_state(). is_active()
returns True only when:
  active AND paused_by is None AND not drawdown_halt_pause

So: even if the operator has resumed, drawdown halt band auto-pauses
the loop. Resume from manual pause does NOT clear drawdown_halt_pause
— that resolves automatically when drawdown_bucket band moves back to
non-halt.
