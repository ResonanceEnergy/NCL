Wave 14CN: NOW/STREAM unbreak — exclude metadata aggregates, raise NOW bar, default-drop YT search, stem-dedup polymarket variants

NATRIX trial-run: "now is broken stream is broken".

Audit findings:
  NOW returned 50 narrative_thread/mandate metadata aggregates
  ("Thread on $AI: 3499 units", "AUTO_TRADER decision sentinel",
  "Analysis failed [<retry>]"). Working_context auto-salience was
  surfacing memory-side bookkeeping into the user-facing lane.

  STREAM returned 5 YouTube clickbait promo cards at the top
  ("HOW I LEARNED PYTHON IN 1 WEEK", "MILLIONAIRE BEFORE 25",
  etc.). The Wave 14CH promo detector is forward-only — it tags
  newly-emitted signals but historical promos already in
  agent_signals.jsonl were never re-filtered.

  After first-pass filter (drop narrative_thread/portfolio/mandate
  sources + Analysis-failed/AUTO_TRADER content prefixes) NOW
  dropped to 6 — but then dropped to 0 when I added too many
  exclusions. Then dropped to 50 google_trends sports topics
  (savannah guthrie / orioles / jarren duran) when I relaxed the
  filter back. So NOW = sports = also broken.

Wave 14CN final shape:

  /intel/now restructure:
    - Skip working_context auto-salience entirely
      (operator-pinned items still get in)
    - Tail-scan agent_signals.jsonl for FOCUSED route signals
    - Composite floor 0.55 → 0.70 so MEDIUM-tier google_trends
      sports drops out (only HIGH/CRITICAL plus ≥0.70 composite)
    - Exclude src startswith: narrative_thread, portfolio:,
      mandate:, *_mandate, auto_trader
    - Exclude content startswith: "Analysis failed",
      "[Position closed]", "AUTO_TRADER"
    - Exclude src_head in (youtube, youtube_search) — same
      noise as STREAM (NATRIX already said curated YTC pipeline
      is the right surface for video content)
    - Read-time _is_promo_text() filter on title + content
      head for pre-fix garbage still in jsonl
    - Stem-dedup per (source, number-normalized-title) so
      polymarket per-outcome variants collapse ("Will Musk
      post 500+ tweets", "80-99 tweets", "100-149 tweets"
      → one card). Operator-pinned items bypass dedup.

  /intel/stream:
    - Default-exclude source_head in (youtube, youtube_search)
      unless user passes ?sources= explicitly (NATRIX called
      them "creator noise that buries real intel"). Curated
      YTC pipeline reports come through source=youtube_council.
    - Read-time _is_promo_text() filter
    - Drop narrative_thread metadata aggregates
    - Same stem-dedup as NOW
