### Open Decisions (Resolution Required)

#### Initial Shipping Mode
**Decision Framework**: Evaluate based on user experience, development velocity, and market readiness.

- **Option A**: Zero custom iOS app (exports + manual + shortcuts)
  - **Pros**: Faster time-to-market, lower complexity, leverages existing iOS capabilities
  - **Cons**: Limited automation, manual intervention required, less integrated experience
  - **Risk Level**: Low technical risk, higher UX friction
  - **Timeline**: 3-6 months to production-ready

- **Option B**: Minimal NCL iOS companion app from day one
  - **Pros**: Native integration, automated collection, superior user experience
  - **Cons**: App store approval process, development overhead, platform dependencies
  - **Risk Level**: Higher technical complexity, regulatory hurdles
  - **Timeline**: 6-12 months to production-ready

**Recommended**: Start with Option A for rapid validation, transition to Option B based on user feedback and market traction.

#### Voice Control Mode
**Decision Framework**: Balance privacy, usability, and technical feasibility.

- **Push-to-Talk**: Explicit activation via button press
  - **Pros**: Maximum privacy, clear user intent, battery efficient
  - **Cons**: Less convenient, requires manual activation
  - **Privacy**: Opt-in only, no background listening
  - **UX**: Explicit control, clear boundaries

- **Wake Word**: Always-listening with keyword activation
  - **Pros**: Hands-free operation, natural interaction, faster response
  - **Cons**: Continuous listening, privacy concerns, battery drain
  - **Privacy**: Local processing only, no cloud transmission
  - **UX**: Seamless integration, ambient computing feel

**Recommended**: Push-to-Talk for initial release to prioritize privacy, with Wake Word as optional advanced feature for power users.
