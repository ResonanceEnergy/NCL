# FRACTAL FUTURE — Gates Checklist (v1.1)

## Purpose
Lightweight but real governance gates applied before any output becomes doctrine or gets published.

---

## Gate FF-1: Definition Clarity
- [ ] Is the pattern/claim operationally defined?
- [ ] Can it be explained in one sentence + one diagram?
- [ ] Is the language precise (not poetic only)?

## Gate FF-2: Evidence / Observation
- [ ] What supports this? (observation, experiment, or strong analogy)
- [ ] Is the evidence level labeled? (E0–E4)
- [ ] What would falsify it (if testable)?

### Evidence Levels
| Level | Description |
|-------|-------------|
| E0 | Conceptual / philosophical |
| E1 | Observational (examples exist) |
| E2 | Correlational (some measurable association) |
| E3 | Experimental (repeatable protocol with data) |
| E4 | Replicated (multiple runs, consistent results) |

## Gate FF-3: Safety & Ethics
- [ ] Any risk of coercion, manipulation, or psychological harm?
- [ ] Any unsafe claims presented as medical advice?
- [ ] Is consent-first framing maintained?
- [ ] Does it avoid weaponized pattern use?

## Gate FF-4: Multi-Scale Coherence
- [ ] Does it remain coherent at micro/meso/macro levels?
- [ ] Does scale introduce contradictions?
- [ ] Is it explicitly bounded if it only works at certain scales?

## Gate FF-5: Regeneration Constraint
- [ ] Does the idea preserve its inputs? (human attention, energy, resources)
- [ ] Does it avoid extraction loops?
- [ ] Is growth sustainable or does it deplete?

## Gate FF-6: Entropy Gate
- [ ] Where does entropy enter this system?
- [ ] Where does entropy accumulate?
- [ ] Where does entropy dissipate safely?
- [ ] What fails first, and does it cascade?
- [ ] What is the bounded maintenance load?
- [ ] Auto-reject if: requires ideal conditions, cascading failure plausible, maintenance superlinear, disorder offloaded onto humans

---

## Quick Pass/Fail (for fast checks)
```

FF-2 Evidence:   PASS / FAIL  (E-level: ___)
FF-3 Safety:     PASS / FAIL
FF-4 Scale:      PASS / FAIL
FF-5 Regen:      PASS / FAIL
FF-6 Entropy:    PASS / FAIL
```
