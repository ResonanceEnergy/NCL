# ENTROPY MAP — FF-EM-002: Human Attention & Cognitive Load

- Domain: human_cognition
- System: Creator's daily attention and decision bandwidth

## Entropy Ingress (Where Disorder Enters)
- Unreliable internet / connectivity loss
- Context switching between NCL projects (agency runtime, onedrop, FRACTAL FUTURE, AAC)
- Parallel initiatives competing for attention
- Notification overload (Telegram, Discord, GitHub, email)
- Incomplete handoffs between work sessions
- Ambiguity / unclear next steps (decision fatigue)

## Accumulation Zones (Where It Piles Up)
- Too many open workstreams without clear priority
- Session context lost between interrupted sessions
- Unfinished tasks that linger (half-built features)
- Documentation debt (things discussed but not recorded)
- Decision backlog (deferred choices compound into confusion)
- Emotional friction from repeated context re-loading

## Dissipation Paths (Where It Exits Safely)
- Session memory saves (snapshot progress before disconnect)
- One-drop formatting (copy-pasteable, works offline)
- Clear daily top-3 priorities (reduce decision surface)
- Doctrine-driven defaults (skip decisions the doctrine already made)
- Agent delegation (let the swarm handle routine tasks)
- Time-boxed deep work blocks (concentrated entropy dissipation)

## Budgets (Thresholds)
- Max active workstreams: 3 (at any moment)
- Max daily context switches: 5
- Max decisions deferred: 3 per week
- Session save interval: every meaningful milestone
- Offline readiness: 100% of outputs must work without internet

## Valves (Explicit Mechanisms)
| Valve | Trigger | Action |
|-------|---------|--------|
| V1 | >3 active workstreams | Park lowest priority, save state |
| V2 | Internet unreliable | Switch to offline-first mode, session save |
| V3 | >5 context switches/day | Lock current workstream for remainder |
| V4 | Decision backlog >3 | Force doctrine-default resolution batch |
| V5 | Session interrupted | Auto-save memory + todo state + next-action |
| V6 | Emotional friction detected | Cooldown: review wins, re-anchor to doctrine |

## Failure Modes (What Fails First)
- Connectivity loss without session save → context evaporation
- Too many parallel initiatives → nothing ships
- Decision fatigue → stall / procrastination
- No daily priority → reactive mode (entropy wins)

## Blast Radius (Local vs Cascading)
- **Goal:** Fail-local by workstream; a stall in one project doesn't collapse another
- **Prevention:** Clear workstream boundaries + session saves + doctrine defaults
- **Worst case:** Full cognitive shutdown → all projects stall (prevent with V6 + daily cadence)

## Actions (What to Change Next)
1. Start every session with Daily Ops Brief (3 priorities, active workstreams, entropy snapshot)
2. End every session with session memory save
3. Use one-drop format for all deliverables (offline-resilient)
4. Apply 3-workstream cap immediately
5. Doctrine-default any deferred decisions older than 1 week
