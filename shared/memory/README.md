# Shared Memory Index

Central index for NCL's institutional memory system.

## Memory Store Location

Runtime memory units are managed by `runtime/memory/store.py` (MemoryStore class) and stored as individual JSON files in:
- `memory-processing/long-term/` — Permanent, indexed knowledge
- `memory-processing/working/` — Active session context
- `memory-processing/decay/` — Fading memories awaiting review

## This Directory

Contains shared memory artifacts that need to be accessible across multiple NCL subsystems:
- `memory-index.json` — Tag-based index for fast recall (generated by consolidation)
- `domain-catalog.json` — Domain hierarchy and memory counts

## Architecture Reference

See `workspaces/memory-processing/references/memory-architecture.md` for the full three-tier memory model documentation.

## Runtime API

The memory system is accessible via the NCL Brain API:
- `POST /memory/store` — Store a new memory unit
- `GET /memory/recall?query={query}` — Search memories
- `GET /memory/stats` — Memory system health metrics
- `POST /memory/consolidate` — Trigger manual consolidation
